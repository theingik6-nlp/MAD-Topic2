Reusable Example Object

Use the sample "Course Enrollment" object below to practise modelling data as JSON, XML, and loading it with AJAX.

Object Description

  • Course: id, title, category, startDate
  • Instructor: firstName, lastName, email
  • Learners: array/list of learner objects (name, progressPercent, completed)
  • Settings: notifications (boolean), deliveryModes (array), maxSeats (number)

JSON Representation

{
  "course": {
    "id": "WD-101",
    "title": "Intro to Web Development",
    "category": "Programming",
    "startDate": "2024-09-01"
  },
  "instructor": {
    "firstName": "Maya",
    "lastName": "Lopez",
    "email": "maya.lopez@example.edu"
  },
  "learners": [
    { "name": "Ali", "progressPercent": 42, "completed": false },
    { "name": "Jin", "progressPercent": 100, "completed": true }
  ],
  "settings": {
    "notifications": true,
    "deliveryModes": ["in-person", "online"],
    "maxSeats": 25
  }
}

XML Representation

<courseEnrollment>
  <course id="WD-101" category="Programming" startDate="2024-09-01">
    <title>Intro to Web Development</title>
  </course>
  <instructor email="maya.lopez@example.edu">
    <firstName>Maya</firstName>
    <lastName>Lopez</lastName>
  </instructor>
  <learners>
    <learner completed="false" progressPercent="42">Ali</learner>
    <learner completed="true" progressPercent="100">Jin</learner>
  </learners>
  <settings notifications="true" maxSeats="25">
    <deliveryModes>
      <mode>in-person</mode>
      <mode>online</mode>
    </deliveryModes>
  </settings>
</courseEnrollment>

AJAX Practice Scenario

Create files:

  • course-enrollment.json containing the JSON above.
  • course-enrollment.xml containing the XML above (optional challenge: fetch and parse XML).
  • ajax-course.html with a button that uses fetch("course-enrollment.json") to load learner progress and render a Bootstrap card or plain list.

AJAX exercise steps:

  1. Show "Loading..." text when the request starts.
  2. Parse the JSON and display the course title, instructor name, and a list of learners with their progress.
  3. Handle errors by showing a dismissible alert.
  4. Bonus: add a toggle to switch between JSON and XML sources and parse accordingly.
